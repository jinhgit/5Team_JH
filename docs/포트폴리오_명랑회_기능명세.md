# 명랑회 (Myeongrang-hoe) — 포트폴리오 기능·기술 명세

> **프로젝트 유형**: 캠퍼스 동행 펀딩 웹 앱 (명지대 해커톤 5Team 기반 확장)  
> **문서 목적**: 구현·수정·추가한 기능을 포트폴리오용으로 빠짐없이 정리  
> **최종 정리 기준**: 포크 `jinhgit/5Team_JH` · 메인 PR [#3](https://github.com/mju-myongjithon/5Team/pull/3)  
> **작업 형태**: FE 로컬 스토어 데모 → **Spring Boot API + FE 서버 연동 + UX 고도화** 전면 확장

---

## 1. 프로젝트 한 줄 소개

**명랑회**는 대학생이 **같은 관심사·근처 위치**로 소규모 모임을 모집·참여하고,  
목표 인원이 모이면 채팅·일정 확정·후기(햇살지수)로 이어지는 **동행 펀딩(크루 모집) 서비스**입니다.

## 2. 문제 정의 · 솔루션

| 문제 | 솔루션 |
|------|--------|

## 4. 시스템 아키텍처

```
┌─────────────────┐     HTTPS/HTTP JSON      ┌──────────────────────┐
│  React (Vite)   │  ◄──────────────────►    │  Spring Boot REST    │
│  - pages/       │     Bearer Token         │  - Controllers       │
│  - store/       │                          │  - Services          │
│  - lib/api.ts   │                          │  - JPA Repositories  │
└────────┬────────┘                          └──────────┬───────────┘
         │                                              │
         │  localStorage 폴백                           │
         ▼                                              ▼
  ┌──────────────┐                            ┌─────────────────┐
  │ FE 로컬 DB   │                            │ H2 File DB      │
  │ (미러/오프라인)│                            │ + /uploads 파일  │
  └──────────────┘                            └─────────────────┘
         │
         ├── Kakao Maps / Share SDK
         ├── Browser Notification
         └── Geolocation
```

### 4.1 FE–BE 연동 전략 (핵심 설계)

1. **토큰 있음** → API 호출 후 로컬 스토어에 미러 (`upsert`)
2. **토큰 없음 / 네트워크 실패** → 로컬 스토어로 데모 가능 (폴백)
3. **AuthBootstrap**으로 앱 기동 시 `GET /api/users/me` 복구
4. **RequireAuth**로 보호 라우트 가드

→ *서버 우선, 오프라인 내성* 하이브리드로 해커톤 데모와 실서비스 중간 지점 확보

## 9. 핵심 구현 하이라이트 (포트폴리오 포인트)

### 9.1 하이브리드 상태 관리
- 서버를 진실 공급원(Source of Truth)으로 두고, FE 스토어는 **캐시/오프라인 미러**
- 네트워크 불안정한 데모 환경에서도 화면이 비지 않음

### 9.2 학교 인증 파이프라인
- OTP 일회성·만료·가입 윈도우
- SMTP 실연동과 개발 모드(코드 노출) 분기
- BCrypt + Bearer 세션 테이블

### 9.3 동행 펀딩 라이프사이클
```
생성 → 참여/찜 → 성사임박 알림 → 인원 확정 or 목표 달성
  → 일정 확정 → 채팅 → 만남 → 후기(햇살지수) → (선택) 신고/차단
```

### 9.4 지도 UX
- 카카오 SDK 로딩 분리, `relayout`, 클러스터, 투어 재생 버튼
- 커버 이미지 없을 때 위치 타일 폴백 (카카오/OSM)

### 9.5 채팅 실사용성
- 폴링 간격 조정, 미읽음 배지, IME 이중 전송 버그 수정
- 읽음 휴리스틱(상대 응답 기반)

### 9.6 발견 경험
- 다중 필터(거리·날짜·요금·카테고리) + AND 검색 + 관심사 맞춤 섹션

8. **후기** — 햇살지수  
9. **공유 / 신고·차단** — 안전 UX

## 13. 구현 과정에서 해결한 이슈

| 이슈 | 해결 |
|------|------|
| 카카오맵 빈 화면 | SDK 로딩·위치 대기 분리, `relayout`, 에러 카피 |
| 채팅/댓글 이중 등록 | 서버 응답 단일 반영 + IME `isComposing` 가드 |
| 커버 이미지 용량 | 2MB 제한, 서버 파일 저장, Lob 한계 회피 |
| H2 NOT NULL 마이그레이션 | default 컬럼 + 수동 보정 |
| 모바일 접속 | FE `0.0.0.0`, API LAN IP, CORS origin pattern |
| OSM 정적맵 불안정 | 타일 폴백으로 안정화 |
